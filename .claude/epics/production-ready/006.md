### Step 2 — Railway setup

3. Add PostgreSQL plugin: Railway dashboard → + New → Database → PostgreSQL
4. Railway auto-sets `DATABASE_URL` — but format is postgres://, need to adapt for Npgsql

### Step 3 — Environment variables in Railway dashboard
```
JWT_SECRET=<generate: openssl rand -base64 48>
ALLOWED_ORIGINS=https://<your-app>.railway.app
ASPNETCORE_ENVIRONMENT=Production
ASPNETCORE_URLS=http://0.0.0.0:$PORT
```

Note: Railway injects `$PORT` automatically. Update `Program.cs` to respect `ASPNETCORE_URLS` env var (it already does by default in ASP.NET Core).

### Step 4 — Auto-migrate on startup
In `Program.cs`, ensure this runs before `app.Run()`:
```csharp
using (var scope = app.Services.CreateScope())
{
    var db = scope.ServiceProvider.GetRequiredService<AppDbContext>();
    db.Database.Migrate();
}
```

### Step 5 — Railway PostgreSQL connection string format
Railway provides `DATABASE_URL` as `postgresql://user:pass@host:port/db`.
Npgsql needs `Host=...;Port=...` format. Add a helper in `Program.cs`:
```csharp
var rawDbUrl = Environment.GetEnvironmentVariable("DATABASE_URL") ?? "";
if (rawDbUrl.StartsWith("postgresql://") || rawDbUrl.StartsWith("postgres://"))
{
    var uri = new Uri(rawDbUrl);
    rawDbUrl = $"Host={uri.Host};Port={uri.Port};Database={uri.AbsolutePath.TrimStart('/')};Username={uri.UserInfo.Split(':')[0]};Password={uri.UserInfo.Split(':')[1]};SSL Mode=Require;Trust Server Certificate=true";
}
builder.Configuration["ConnectionStrings:DefaultConnection"] = rawDbUrl;
```
