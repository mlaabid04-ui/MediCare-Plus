# Task: Refactor monolithic files into per-class files

## Description

Split `Controllers.cs` and `OtherServices.cs` into individual files — one class per file — to make the codebase navigable and maintainable. This is a pure structural change with zero logic modifications.

Files to split:

**HospitalApp.API/Controllers/Controllers.cs** → split into:
- `AuthController.cs`
- `DoctorController.cs`
- `AppointmentController.cs`
- `PatientController.cs`
- `NotificationController.cs`
- `ChatController.cs`
- `AdminController.cs`

**HospitalApp.API/Services/OtherServices.cs** → split into:
- `DoctorService.cs`
- `PatientService.cs`
- `AdminService.cs`
- `ChatService.cs`

Also verify `AuthService.cs`, `AppointmentService.cs`, `NotificationService.cs` are already individual files (they are — leave them untouched).

## Acceptance Criteria
- [ ] Each controller class lives in its own `.cs` file under `Controllers/`
- [ ] Each service class lives in its own `.cs` file under `Services/`
- [ ] Original `Controllers.cs` and `OtherServices.cs` are deleted
- [ ] `dotnet build HospitalApp.API` passes with zero errors
- [ ] No logic, method signatures, or namespaces changed — only file organization

## Technical Details

- Read `Controllers.cs` fully, identify each `[ApiController]` class boundary
- For each class: create new file, paste class (with all using statements and namespace), save
- Repeat for `OtherServices.cs` — identify each `IXxxService` interface + `XxxService : IXxxService` pair
- Keep the same namespace (`HospitalApp.API.Controllers` / `HospitalApp.API.Services`)
- After splitting, delete the two original monolithic files
- Run `dotnet build` to confirm no broken references

None — this is the foundation task. All other tasks build on this clean structure.
