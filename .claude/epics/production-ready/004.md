# Task: CORS hardening

## Description

Replace the current `AllowAll` CORS policy with an explicit origin allowlist. In development, allow localhost. In production, read allowed origins from the `ALLOWED_ORIGINS` environment variable.

## Acceptance Criteria
- [ ] `AllowAll` CORS policy is removed from `Program.cs`
- [ ] A request from an unlisted origin receives a CORS error (browser blocks it)
- [ ] `http://localhost:5001` is allowed in development
- [ ] Production allowed origins are read from `ALLOWED_ORIGINS` env var (comma-separated)
- [ ] SignalR hubs still work (CORS must allow credentials for SignalR)
- [ ] `dotnet build` passes

## Technical Details

Replace in `Program.cs`:
```csharp
// REMOVE this:
builder.Services.AddCors(options =>
{
    options.AddPolicy("AllowAll", policy =>
        policy.AllowAnyOrigin().AllowAnyMethod().AllowAnyHeader());
});

// ADD this:
var allowedOrigins = (Environment.GetEnvironmentVariable("ALLOWED_ORIGINS") ?? "http://localhost:5001")
    .Split(',', StringSplitOptions.RemoveEmptyEntries | StringSplitOptions.TrimEntries);

builder.Services.AddCors(options =>
{
    options.AddPolicy("AllowConfigured", policy =>
        policy.WithOrigins(allowedOrigins)
              .AllowAnyMethod()
              .AllowAnyHeader()
              .AllowCredentials());  // Required for SignalR
});
```

And update `app.UseCors("AllowAll")` → `app.UseCors("AllowConfigured")`.

Update `.env.example` to document `ALLOWED_ORIGINS`.
