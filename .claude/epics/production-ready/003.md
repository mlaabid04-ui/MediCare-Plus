# Task: JWT refresh token flow

## Description

Implement a proper token refresh mechanism. Access tokens become short-lived (15 min). A refresh token (30 days) is issued at login and can be exchanged for a new access token. This allows session continuity without long-lived access tokens, and enables logout/revocation.

## Acceptance Criteria
- [ ] `POST /api/auth/login` returns `{ accessToken, refreshToken, expiresIn }`
- [ ] `POST /api/auth/register` also returns `{ accessToken, refreshToken, expiresIn }`
- [ ] Access token expires in 15 minutes
- [ ] `POST /api/auth/refresh` with valid refresh token returns new `{ accessToken, expiresIn }`
- [ ] `POST /api/auth/refresh` with expired or revoked token returns HTTP 401
- [ ] `POST /api/auth/logout` revokes the refresh token and returns 200
- [ ] `RefreshTokens` table exists in DB with a migration
- [ ] Mobile app auto-calls `/api/auth/refresh` when it gets a 401, then retries the original request
- [ ] Mobile app calls `/api/auth/logout` on user logout and clears stored tokens

## Technical Details

### New DB entity (in `AppModels.cs` / `AppDbContext.cs`):
```csharp
public class RefreshToken
{
    public Guid Id { get; set; }
    public string UserId { get; set; }
    public User User { get; set; }
    public string Token { get; set; }         // store SHA-256 hash
    public DateTime ExpiresAt { get; set; }
    public DateTime CreatedAt { get; set; }
    public bool IsRevoked { get; set; }
    public DateTime? RevokedAt { get; set; }
}
```

### AuthService changes:
- `GenerateRefreshToken()`: create 64-byte random token, return raw value, store SHA-256 hash
- `RefreshAccessToken(string rawToken)`: hash input, find in DB, check expiry + revoked, return new access token
- `RevokeRefreshToken(string rawToken)`: mark as revoked

### New endpoints in `AuthController.cs`:
```
POST /api/auth/refresh   body: { refreshToken: string }
POST /api/auth/logout    body: { refreshToken: string }   [Authorize]
```

### Mobile `ApiService.cs` changes:
- Store both `accessToken` and `refreshToken` in Preferences
- Add retry logic: on 401, call `/api/auth/refresh`, update stored token, retry original request once

### Migration:
```bash
dotnet ef migrations add AddRefreshTokens
dotnet ef database update
```
