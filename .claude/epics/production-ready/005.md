# Task: HTTPS configuration and remove mobile SSL bypass

## Description

Enable HTTPS on the backend and remove the dangerous SSL certificate validation bypass from the mobile client. When deployed to Railway, HTTPS is provided automatically. For local development, use the .NET dev certificate.

## Acceptance Criteria
- [ ] `HttpClientHandler` with `ServerCertificateCustomValidationCallback = (_, _, _, _) => true` is removed from mobile `ApiService.cs`
- [ ] Mobile client uses the default `HttpClient` (trusts system/OS certificate store)
- [ ] Backend `appsettings.json` includes an HTTPS endpoint configuration
- [ ] `dotnet dev-certs https --trust` documented in README/setup notes
- [ ] Mobile app in Debug mode can connect to local dev server over HTTPS
- [ ] Mobile app in Release mode connects to Railway HTTPS URL (tested in Task 006)

## Technical Details

### Mobile `ApiService.cs` — remove SSL bypass:
```csharp
// REMOVE:
var handler = new HttpClientHandler
{
    ServerCertificateCustomValidationCallback = (message, cert, chain, errors) => true
};
_httpClient = new HttpClient(handler);

// REPLACE WITH:
_httpClient = new HttpClient();
_httpClient.BaseAddress = new Uri(AppConfig.ApiBaseUrl);
_httpClient.Timeout = TimeSpan.FromSeconds(30);
```

### New file: `HospitalApp.Mobile/AppConfig.cs`:
```csharp
namespace HospitalApp.Mobile;

public static class AppConfig
{
#if DEBUG
    public const string ApiBaseUrl = "https://localhost:5001/api/";
    public const string HubBaseUrl = "https://localhost:5001";
#else
    public const string ApiBaseUrl = "https://<your-app>.railway.app/api/";
    public const string HubBaseUrl = "https://<your-app>.railway.app";
#endif
}
```

Update all hardcoded URL references in `ApiService.cs` and `SignalRService.cs` to use `AppConfig.ApiBaseUrl` and `AppConfig.HubBaseUrl`.

### Backend `appsettings.json` — add HTTPS endpoint:
```json
"Kestrel": {
  "Endpoints": {
    "Http": { "Url": "http://0.0.0.0:5001" },
    "Https": { "Url": "https://0.0.0.0:5443" }
  }
}
```

### Dev setup note:
```bash
dotnet dev-certs https --trust
```

Android emulator note: Android does not trust .NET dev certs by default. For local Android testing, network security config or a proper cert is needed. Railway HTTPS (Let's Encrypt) is trusted by Android automatically.
