# Task: Secrets management via environment variables

## Description

Remove all hardcoded credentials from source files and replace with environment variable references. This eliminates the risk of secret leakage via source control.

**Current hardcoded values to remove:**
- `appsettings.json`: `"Password=123456789"` in connection string
- `appsettings.json`: `"Key": "HospitalSecretKey2024!@#$VeryLongAndSecure"` in JWT config

## Acceptance Criteria
- [ ] `appsettings.json` contains no passwords, secrets, or credentials
- [ ] `appsettings.json` uses placeholder values (empty strings or `"<set via env var>"`)
- [ ] `Program.cs` or config binding reads `DATABASE_URL` env var for connection string
- [ ] `Program.cs` or config binding reads `JWT_SECRET` env var for JWT signing key
- [ ] App throws a clear startup error if `JWT_SECRET` is missing or too short (<32 chars)
- [ ] `.env.example` file created at solution root with all required vars documented
- [ ] `.gitignore` at solution root includes `.env`
- [ ] `dotnet build` and `dotnet run` (with env vars set) still work

## Technical Details

In `appsettings.json`:
```json
{
  "ConnectionStrings": {
    "DefaultConnection": ""
  },
  "Jwt": {
    "Key": ""
  }
}
```

In `Program.cs`, after `builder.Configuration` is available:
```csharp
// Override with env vars
var dbUrl = Environment.GetEnvironmentVariable("DATABASE_URL");
if (!string.IsNullOrEmpty(dbUrl))
    builder.Configuration["ConnectionStrings:DefaultConnection"] = dbUrl;

var jwtSecret = Environment.GetEnvironmentVariable("JWT_SECRET")
    ?? throw new InvalidOperationException("JWT_SECRET environment variable is required");
builder.Configuration["Jwt:Key"] = jwtSecret;
```

`.env.example`:
```
DATABASE_URL=Host=localhost;Port=5432;Database=HospitalDB;Username=postgres;Password=yourpassword
JWT_SECRET=your-very-long-secret-key-at-least-32-characters
ALLOWED_ORIGINS=http://localhost:5001
```
